  • mappingCandidates(for:withModel:at:)

    Returns mappings candidates of correct viewType, for which modelTypeCheckingBlock with model returns true.

    Note

    Usually returned array will consist of 0 or 1 element. Multiple candidates will be returned when several mappings correspond to current model - this can happen in case of protocol or subclassed model.

    See also

    addMappingForViewType(_:viewClass:)
